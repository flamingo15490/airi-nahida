# 当前架构现状

## 当前总体架构
目前系统不是一个完全内聚的单体 AIRI，而是：
- **AIRI 主程序**：负责主体交互、UI、角色驱动、工具接口与运行框架
- **外部语音系统**：GPT-SoVITS + TTS Adapter
- **外部绘图系统**：ComfyUI / 兼容适配层
- **外部记忆系统**：`D:\AIRI-Memory`
- **外部主动陪伴系统**：proactive companion / sidecar

因此，当前架构是 **AIRI + 外部本地服务协作**，不是“一切都已经内置到 AIRI 里”。

## 当前需要保持外部独立的部分
第一阶段中，以下部分默认继续保持外部运行：
- GPT-SoVITS
- TTS Adapter
- ComfyUI / 图像服务
- `D:\AIRI-Memory`
- proactive sidecar
- 现有一键启动体系

这些服务目前先以“兼容”为目标，而不是立刻内置重构。

## 已知源码层问题
### 1. Context bridge 问题
此前出现过与 `structuredClone`、上下文广播、不可克隆对象相关的问题。
这类问题会影响：
- 对话消息桥接
- 视觉上下文传递
- 图片/上下文消息提交稳定性

当前方向是把已验证有效的净化/清洗逻辑迁移进源码，而不是继续依赖临时本地补丁。

### 2. 视觉系统结构问题
已确认：视觉系统此前“必须保持设置页或开发者页常开才能继续捕获”，不是简单配置问题，而是源码结构问题。

已知根因：
- 实际驱动 ticker / capture loop 的逻辑原本放在 `devtools/vision.vue`
- 页面卸载时会显式停止 ticker 和 stream
- 所以视觉运行被错误地绑定在页面生命周期上

正确方向应为：
- App 或更高层级负责长生命周期 vision runner
- 设置页负责配置
- devtools 页面负责调试、预览与控制
