- 与当前阶段能力、验收口径、中文化要求最相关的入口，优先看 [已实现功能](./已实现功能.md)。
- 如果是新开子进程做补尾、汉化、验收或收口，先看 [阶段收尾结论](./06-阶段收尾结论.md)。
- 这些文件已经统一收纳到 `project-docs/`，避免仓库根目录继续堆散文档。
- 文档内容应始终与最近一次通过验收的自定义源码基线保持一致。
