# 最终愿景与阶段路线图

## 最终愿景
最终希望形成的，不是一个零散拼接的工具集合，而是一个：
- 以 AIRI 为核心
- 与本地语音、视觉、绘图、记忆、主动陪伴能力深度协同
- 风格贴近“纳西妲”
- 适合长期使用
- 可持续维护和扩展
的定制化桌面陪伴系统。

## 最终形态关键词
最终理想形态大致包括：
- 稳定的对话主链路
- 自然的人格表现
- 长期记忆与自然回忆
- 稳定视觉理解
- 可用且轻量的本地生图
- 自然的本地语音输出
- 克制但拟人的主动行为
- 结构清晰、便于后续继续开发

## 已完成阶段概况

### 第一阶段：源码稳定化与关键补丁迁移
已基本完成。

已落地重点：
- context bridge 净化修复
- vision 常驻运行修复
- 最小结构收尾与基础验证

阶段结果：
- 源码版已具备更稳定的桥接基础
- vision 已不再依赖设置页 / devtools 页常开
- 可以更放心地进入后续本地协同深化

### 第二阶段：本地能力协同稳定化
已基本完成主体落地。

已落地重点：
- sidecar / memory 的桌面端独立配置边界
- 本地协同设置入口
- 原有本地配置与数据复用能力

阶段结果：
- 源码版 AIRI 已不再只是“能跑”
- 已具备更正式的本地协同接入边界

### 第三阶段：角色定制深化
已完成第一轮人格层落地。

已落地重点：
- 纳西妲人格 supplement
- 三档模式
- 人格层持久化配置
- reminder / proactive tone 的人格统一

阶段结果：
- AIRI 已能较稳定地“像纳西妲地说话”
- 角色化表达进入可持续细修阶段

### 第四阶段：外部记忆闭环与受控写回
已完成第一轮闭环落地。

已落地重点：
- `D:\AIRI-Memory` 正式接入边界
- 记忆读取与 prompt supplement
- 受控写回
- 稳定性 heuristics 与写回决策可解释性

阶段结果：
- AIRI 已具备“读外部记忆 + 少量高价值写回”的基础能力
- 记忆层已经进入稳定深化状态

### 第五阶段：受控主动陪伴协同层
已完成治理层落地，但仍保留已知限制。

已落地重点：
- proactive companion 治理层
- 事件筛选 / 冷却 / 去重 / 展示强度
- 历史解释能力
- 治理看板

阶段结果：
- 系统已从“黑箱没反应”提升成“收到事件后可解释为什么没反应”
- 当前主要剩余问题是 sidecar 真实事件样本不足，导致高频实体验证反馈慢

### 第六阶段：协同判断层
已完成第一轮结构化收口。

已落地重点：
- companion coordination 共享契约与主进程 manager
- 主聊天与主动提醒接入统一 coordination snapshot
- 协同总览页
- 统一的 reason / surface / snapshot 口径

阶段结果：
- 记忆、人格、主动陪伴三条线已经不再只是平行堆放
- AIRI 已具备一套更明确的协同判断中间层

### 第七阶段：系统健康与日用运维收口
已完成第一轮日用运维收口。

已落地重点：
- system health 共享契约、主进程 manager 与 renderer store
- system health 设置页与 settings 首页快速摘要
- App 级自动 startup diagnostics
- 启动失败顶部提示条
- `start-airi-full.bat` 统一启动脚本

阶段结果：
- 日常使用时，已经更容易判断“当前到底哪里正常、哪里异常”
- 源码版的日用维护成本较前几个阶段进一步下降

## 当前最现实的路线
当前最合理的路线不是“推倒重来”，而是：
1. 先收口前七阶段的阶段成果
2. 整理当前基线与验收结论
3. 再推进记忆、人格、主动陪伴三者的细腻协同
4. 最后才考虑更大幅度的结构升级或独立化

## 当前阶段之后应进入什么状态
前七阶段收口后，理想状态应为：
- 源码版具备较稳定的日用基础
- 能较稳定地验证记忆、人格、主动陪伴三条主线
- 能较快定位当前健康状态与运维问题
- 可以基于更清晰的结构推进下一阶段深化

## 下一阶段最自然的入口
当前最自然的下一阶段入口不是继续铺更多基础设施，而是：
- 提升记忆写回判定质量
- 提升纳西妲人格表达一致性
- 提升主动陪伴与记忆 / 人格的协同细腻度
- 继续降低日常使用中的维护成本与配置成本
- 继续提升“问题出现时为什么这样”的可解释性
