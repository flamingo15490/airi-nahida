# 项目总目标

## 这个项目是什么
这个项目的目标，不是继续无限叠加本地热补丁，而是在 `D:\airi-source\airi` 这套 AIRI 开源源码基础上，逐步整理出一个更适合当前本地使用环境的 **AIRI 定制开发版**。

它的长期方向是：
- 保留 AIRI 作为主体框架
- 保留当前已经验证可用的本地外部服务体系
- 把真正关键、会长期影响体验的修复迁移进源码
- 为后续“纳西妲专用陪伴型智能体”打下稳定基础

## 最终目标
最终希望做成一个：
- 适合长期桌面陪伴使用
- 具有稳定语音、视觉、绘图、记忆与主动性能力
- 角色风格贴合“纳西妲”
- 可维护、可迭代、可回退
- 不依赖大量脆弱手工补丁的 AIRI 定制版本

## 当前阶段目标
当前还处于 **第一阶段：源码稳定化与关键补丁迁移**。

这个阶段的重点不是“做完最终形态”，而是：
1. 收敛已经验证有效的关键修复
2. 解决影响体验的结构性问题
3. 保持现有外部体系可继续工作
4. 给后续阶段留出干净的扩展入口

## 当前阶段最优先事项
按优先级排序：
1. 修复并稳定 context bridge / structuredClone / 非可克隆对象污染问题
2. 修复视觉系统只能在设置页或开发者页常开时才工作的结构性问题
3. 把视觉运行机制改为 App 级常驻，而不是页面级生命周期
4. 保证与当前外部 sidecar、记忆、语音、生图体系兼容
5. 在不扩大范围的前提下，为下一阶段留出扩展空间

## 当前阶段完成标志
如果第一阶段完成，应该达到这些状态：
- context bridge 相关问题在源码层稳定
- 视觉系统不再依赖设置页/开发者页常开
- 源码版 AIRI 至少具备进入实际运行验证的基础
- 当前外部服务体系没有被破坏
- 后续 Codex 或开发者可以在清晰结构上继续迭代
